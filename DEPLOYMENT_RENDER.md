Render Deployment Notes

Public URL
- Deploying on Render gives you a public URL that users in other cities can open.

Current Blueprint
- Service type: web
- Runtime: Docker
- Health check path: /healthz
- Container command binds to Render PORT via Dockerfile

Environment Variables
- FLASK_SECRET_KEY: generated by Render
- APP_ENV: production
- APP_USERNAME: set in Render dashboard
- APP_PASSWORD: set in Render dashboard
- SESSION_COOKIE_SAMESITE: Lax

Persistent Uploads (important)
- The app writes uploaded files to /app/uploads.
- On free instances, local container filesystem is ephemeral and can reset.
- To persist uploads, attach a Render Disk (paid plans):
  1. Open the service in Render dashboard.
  2. Go to Disks.
  3. Add disk and set mount path to /app/uploads.
  4. Redeploy the service.

Verification Checklist
- /healthz returns status ok.
- /login loads from public Render URL.
- Upload a file and confirm it appears under /app/uploads in runtime.
